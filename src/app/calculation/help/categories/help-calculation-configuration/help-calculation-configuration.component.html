Overview

The Calculation Builder produces the relevant steps to build a calculation this will consist of the inputs used in the calculation, the configuration of the steps and the functions to build the calculation. Then the outputs that will be made available from the calculation after this has been run.



Building Calculation

The configuration is done row by row; the calculation is built up by using a combination of functions which need to correspond with each other to build up the calculation. Principally the way this is executed in the application is to create rows of a specific data type which can then be referenced at various points in the configuration to utilise those values within the functions and reuse the values. In order to do this the user can create inputs in the Calculation Inputs, to define these variables within the Calculation Configuration which can then be used e.g. DOB (Date of Birth). The Calculation Configuration then takes these inputs and manipulates them to produce some new values, which then pass into the Calculation Output. The Calculation Output selects various Calculation Configuration values and uses them to output a list of values.

Using Chips

On each function in the Calculation Configuration builder the user will able to drag a set of chips which should be previously set values for example Date of Birth to use within that function to manipulate this. Say for instance weâ€™ve created two input fields for Date of Birth and Date of Marriage in the Input category; these can be amended for each calculation that is run with a value that the user can add in. For a particular function we require to find the date duration between these two dates, so the user will create a new function run for Date Duration. The function requires two date field to be added in, the user can either add in a date value in here by typing this in, alternatively, the user can reference the previously set value for Date of Birth/Date of Marriage as a Chip within the function which can be dragged into the input fields for use. The calculation will then replace the value of Date of Birth/Date of Marriage with the date value that has been assigned to it. In the Maths and IfLogic Functions the chip can be double clicked to add to the end of the formula.



Each function creates an output; the accumulation of all of these outputs creates the calculation in full. A reasonable variable name should be used in all cases in order to reference any changes in the future. Variables names can be reused and overwritten within the calculation later on bare this in mind when configuring the calculations.

Side Navigation

On click of the three rows icon at the top left of the builder the Side Navigation panel will be displayed to the user. Please see below description of all actions.

  • Calculate - Performs calculation of the configuration
  • Save - Initiates the save function of the calculation for use at a later date
  • Test Management - Enables the user to create tests to ensure that the calculation is still performing the correct results on a set of example calculations
  • Release Management - Enables the user to release the calculation for use by a wider audience the additional users will not be able to reconfigure the calculation but to run the end result which will be the released version of this calculation.
  • User Management - Enables the user to define who can access the released calculation
  • Delete Calculation - Perfoms the delete function on the calculation to remove this from the dashboard
  • Exit - Navigates the user back to the dashboard

Top Bar Buttons

On the top row of the builder there are some shortcut icons available to the user, the actions are in the following order: Calculate - Save - Exit



Overview

The Calculation Input section defines all inputs used within the calculation that can be changed by a user based on the reasons for running the calculation. These inputs are then used within the calculation configuration section to configure the calculation.



Columns

Please see below details of all of the columns within this table.

  • Name - The Name of the input which can be used to pick up as part of the calculation configuration
  • Data - The Data type of the input to ensure that the input value is correct going into the calculation, validations will appear if the incorrect data is input
  • Input - The value of the input to be used in the calculation
  • Drop Down List? - If the input should be a drop down list of values for example Yes/No then select true in this field
  • Drop Down Value - If the input is a drop down list of values then please put a comma seperated list of the values needed to be in the drop down
  • Required - If the input is required to make the calculation work then select true in this field and validations will check if this is populated before allowing the user to calculate.

Overview

The Calculation Configuration section is for the user to create a suite of functions for the calculation to manipulate the inputs to calculate the outputs. On adding a new row the user must select a Name and Data type and Function in order to do the configuration. In order to access the function the user needs to click a row and the function will be displayed below the row.



Columns

Please see below details of all of the columns within this table.

  • Group - The name of the group this is to help organise the configuration
  • Function - The name of the function that the row will use, please see help on calculation functions to understand what each of the functions should do.
  • Name - The Name of the row which can be used to pick up as part of the Calculation Output
  • Data - The Data type of the row
  • Output - The value of the output field after the calculation has been run

Overview

The Calculation Output section is for the user to create a suite of outputs from the calculation that will be presented to the user after the calculation has been run successfully. On adding a new row the user must select a Name and Data type which will allow the user to select a Name from the Calculation Configuration menu and will then use this as a value for the output.



Columns

Please see below details of all of the columns within this table.

  • Name - The Name of the output which can be used to pick up as part of the Calculation Configuration
  • Data - The Data type of the output to ensure that the Output can pick up a date value from the Calculation Configuration
  • Variable - The name of the row the output will use from the Calculation Configuration
  • Output - The value of the output field after the calculation has been run
  • Expected Results- This will be used for testing the output is as expected for this calculation
  • Pass - This will show as true if the output is equal to the expected result, otherwise it will show false

Overview

On calculation the system will check that all rows have passed validation before proceeding to calculate this according, this is to ensure that the calculation will successfully run and not error. Once the calculation button has been clicked a popup will show that the calculation is in progress.

Validation

The builder contains various validations to ensure that the calculation is run successfully. In the event of a calculation error the row in the builder that has been affected will display as red and must be clicked in order to show the validation error message.

Success

On successfully calculation a further pop up will be shown to the user to inform that the calculation has been successfully run and the new figures will be displayed to the user.

Overview

On the builder there are various functions to allow the user to interact with the builder.

Add and Delete Rows

The user can Add (Plus Icon) Delete (Cross Icon) the rows on the builder if required. The user must highlight the relevant row in the Calculation Input, Calculation Configuration, Calculation Output and select the relevant place to Add the row or Delete the Row.

Rows - Filtering

The user can filter the rows based on a criteria on the top row under the column headings there is an input field, the user should type in the text of the required filter and the relevant tables will filter based on these values.

Rows - Movable

The user can move each row within the builder by clicking the grid of dots in the first column on each row to move them up or down.
