Overview

The Date Adjustment function enables the user to manipulate dates.

Add/Subtract

The Add and Subtract sub functions allow the user to adjust the date by a period of time. On selecting the Adjustment type as this the user will be shown the below form. The Start Date should be the date to be adjusted, the Period Adjustment section shows what period you would like to adjust date by. Period Type options are below these can be hard coded or a variable previously created for the period which should be a decimal value: Years Days – This value should be a decimal to 3dp the 3dps equate to the days and the full value is the years. Years Months – This value should be a decimal to 2dp the 2dp equate to the months and the full value is the years. Years – This value should be an integer which will be the years. Months – This value should be an integer which will be the month. Days – This value should be an integer which will be the days.

Earlier/Later

The Earlier and Later sub functions will return the earlier of the two dates added into the functions. If either of these options are selected the below screen will be displayed. Adjust The Adjust sub function will adjust a date to a day of the year on or after the date that is input. The Adjustment to be made section will appear on selecting the Adjust sub function this will display some additional fields. The Select Day and Month define the new day and month once the adjustment has been made. The Adjustment defines what direction to adjust the date; the field displays the options defined below: Before / On or Before – Before the date input into the function this could be the same year or the year prior if the date adjustment is prior to the current date that is in the function After / On or After – After the date input into the function this could be the same year or the year after if the date adjustment is after to the current date that is in the function Same Year – Will always return the date adjustment in the same year

Dates Between Two Dates

This function creates an array of the defined date occurrences between two dates. i.e Number of 6th April occurrences between two dates. On selecting this Adjustment Type the below view will be shown, the Start Date and End Date are the criteria of the two dates and Select Day and Select Month will find the dates occurring between these two dates. This will then display an array in the Output on the row as below.

Months Between Two Dates

This function creates an array of the defined date occurrences between two dates with the day being on the same day for each month in the array. This will count the number of months between the two dates and will create an array value for each of these months.

This function allows the user to set a date using the day and month to a date nearest to the date input in the direction required.

This function allows the user to set a date using the day to a date nearest to the date input in the direction required.

This function enables the calculation to lookup a factor table to return the value. The Factors lookup function is dynamic form that will be adjusted based on the selection of the fields. Definitions of the fields are shown below.

Table Name

The name of .CSV file on the factor tables available on the server, see the Factor Maintenance section to find a list of available factor tables to use. The table name should be the name of the file minus the .CSV.

Lookup Type

The vertical lookup data type on the .CSV that the function will be looking up in the first column.

Lookup Value

This is the value to lookup in the vertical column

Output Type

This is the output data type to define the return data type from the factor table If the factor table can be looked up based on the first column only in the .CSV and the user knows which column to be returned, then set to Row Match Required? to False, then add in the column required to be returned. If the user selects a Lookup Type of Decimal the option to Interpolate Years and Months will be shown to allow the lookup to interpolate the return values. If the user does not know which column to return and bases the return column on matching a value in a row to find the column to return then select Row Match Required? To True. This will then reveal some new fields outlined below

Row Match Row Number

Defines which row to use to lookup the value to find the column

Row Match Lookup Type

Defines the data type to lookup the value to find the column

Row Match Value

Defines the value to lookup to find the column

Overview

This function allows the user to build mathematical formula in a dynamic way, a row can be created for each part of the formula, the variables used can be hard coded or the variable names from the builder. Rows can also be deleted if required. If a new row is created Logic in the final column must be populated to build the formula for the next row. Rounding up or down and to a maximum of 6 decimal places can be applied but only on the final row of the formula. Validations will be shown in the maths builder if the formula built doesn’t meet these requirements these will be shown on click of OK and resolved on click of OK. If there is a row added and the previous row doesn’t have a logic value this requires a logic adding in. If there is Logic on the last row and no rows underneath the following validation will be shown. The brackets will also be validated to ensure that the brackets are not left open or closed. Due to the way that the dynamic inputs on multiple lines work when building a formula, the user is required to input a value in Input 1 and Input 2 fields on each row. If the formula goes on to a different row and the formula is required to add 1 on the second row, an input must be made in both input fields. In order to do this you need to add a value of 1 in Input 1 then + and add a value of 0 to Input 2. If 0 wasn’t added in then there will be a validation message that asks for a required field.

Adjustment Types

Years Days – This value would return a decimal to 3dps the 3dps equates to the days and the full value is the years. Years Months – This value would return a decimal to 2dps the 2dps equates to the months and the full value is the years. Years Days (Decimal 6dp) – This value would return a decimal to 6dps which will be days divided by 365 Years Months (Decimal 6dp) – This value would return a decimal to 6dps which will be months divided by 12 Years – This value return an integer which will be the years. Months – This value return an integer which will be the months. Days – This value return an integer which will be the days.

Rounding

The Days in Year allows a different number of days in the year to be used as the yearly value options of these are below: 365 – 365 days per year 365.25 – 365.25 days per year 366 – 366 days per year
